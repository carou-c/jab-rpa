JabRpaServer

Manages the lifecycle of the jab-rpa-server.exe subprocess.

Spawns the 32-bit Rust gRPC server, waits for it to report readiness, and provides clean shutdown.

Can be used as a context manager:

with JabRpaServer() as server:
    ...

__init__ 

__init__(
    *,
    server_path: Path = _SERVER_PATH,
    server_timeout: int = _WAIT_FOR_SERVER_TIMEOUT,
    step: int = _INIT_SERVER_STEP,
) -> None

Configure the server process settings.

Parameters:
  • server_path (Path, default: _SERVER_PATH ) –

    Path to the jab-rpa-server.exe binary.

  • server_timeout (int, default: _WAIT_FOR_SERVER_TIMEOUT ) –

    Maximum seconds to wait for the server to start.

  • step (int, default: _INIT_SERVER_STEP ) –

    Seconds between readiness checks.

start 

start() -> None

Launch the server subprocess and wait for it to become ready.

Reads stdout in a background daemon thread and watches for the server's "listening" message.

Raises:
  • ServerStoppedError –

    If the process exits before reporting readiness.

  • TimeoutError –

    If the server does not become ready within the timeout.

stop 

stop() -> None

Terminate the server subprocess.

Bases: Exception

Raised when the JAB gRPC server process exits before it starts listening.
